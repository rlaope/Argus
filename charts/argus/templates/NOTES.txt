  Argus is a JVM agent — it does NOT deploy pods.
  Follow these steps to instrument your application:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. LABEL YOUR PODS
   Add the following label to pods you want Argus to monitor:

     argus.io/enabled: "true"

   This enables the ServiceMonitor to discover and scrape them.

2. ADD THE ARGUS AGENT TO YOUR JVM APPLICATION

   Option A — Java agent (any JVM app):
     Download the agent JAR and mount it into your pod, then add to your
     container's JAVA_TOOL_OPTIONS or via an init container:

       env:
         - name: JAVA_TOOL_OPTIONS
           value: "-javaagent:/opt/argus/argus-agent.jar $(ARGUS_JAVA_OPTS)"
         - name: ARGUS_JAVA_OPTS
           valueFrom:
             configMapKeyRef:
               name: {{ include "argus.fullname" . }}-config
               key: ARGUS_JAVA_OPTS

   Option B — Spring Boot Starter:
     Add the dependency to your pom.xml or build.gradle:

       # Maven
       <dependency>
         <groupId>io.github.rlaope</groupId>
         <artifactId>argus-spring-boot-starter</artifactId>
         <version>{{ .Chart.AppVersion }}</version>
       </dependency>

       # Gradle
       implementation 'io.github.rlaope:argus-spring-boot-starter:{{ .Chart.AppVersion }}'

3. EXPOSE THE METRICS PORT
   Argus serves metrics on port {{ .Values.agent.port }}. Ensure your Service exposes
   this port with the name "argus-metrics":

     ports:
       - name: argus-metrics
         port: {{ .Values.agent.port }}
         targetPort: {{ .Values.agent.port }}

4. VERIFY PROMETHEUS DISCOVERY
{{- if .Values.serviceMonitor.enabled }}
   A ServiceMonitor has been created. Verify Prometheus picks it up:

     kubectl get servicemonitor {{ include "argus.fullname" . }} -n {{ .Release.Namespace }}

   In Prometheus UI, check Status > Targets for your application pods.
{{- else }}
   ServiceMonitor is disabled. Enable it with:

     helm upgrade {{ .Release.Name }} argus/argus --set serviceMonitor.enabled=true
{{- end }}
